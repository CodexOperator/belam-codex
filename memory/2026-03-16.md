# 2026-03-16 — Session Log

## Early Morning (~03:08 UTC)
- Shael investigated V4 notebook performance — parallel threaded workflow running slower than expected
- Root cause: `batch_size=2048` + 12 concurrent workers causes GPU thrashing on Tesla T4
- Key findings:
  - Optimal worker count = 2 (GIL contention kills throughput beyond that)
  - Batch size 512→4096 gives 3.6× speedup (the real throughput lever)
  - `shuffle=True` on GPU TensorDataset: only ~6% overhead, acceptable
  - `gpu_cleanup()` / `torch.cuda.empty_cache()` inside fold loops acts as global thread barrier — move to end of cell
- Final optimized config: `batch_size=4096`, `n_parallel=2`, cleanup only after loop
- Final execution time: 162.32s (~0.11 fold-runs/sec)
- Created lesson: `lessons/gpu-parallel-thrashing-t4.md`
