# Memory Entry

**2026-03-21T05:14:20Z** · `insight` · importance 4/5

Deep architectural discussion with Shael on Codex Engine V2+ direction. Key conclusions: (1) Modes should be primitives, not flags — orchestration, extend, create, edit all get their own coordinates and namespace. (2) -x extend mode is the meta-layer (lathe builds lathe) for adding new categories, custom code, namespace prefixes. (3) Plugins have presence (persist across turns, hook lifecycle) vs commands are fire-and-forget — different primitive types. (4) Temporal interaction idea: LLMs that play against time rather than humans, using SpacetimeDB for event sourcing, daemon-free via checkpointed state. (5) Persistent agent model: agents spawn with context and keep running rather than per-message cycles. New tasks created: temporal-interaction-llm-gaming, codex-engine-v2-modes-mcp-temporal.
