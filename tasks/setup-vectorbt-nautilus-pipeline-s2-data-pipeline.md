# S2: Data Pipeline — Crypto OHLCV Ingestion into DuckDB

## Scope

- Build a Polars-based crypto OHLCV data ingestion pipeline fetching from free APIs (Binance public preferred)
- Store normalized data in DuckDB with Parquet-native storage
- Provide loader utilities that output data in both VectorBT and NautilusTrader formats
- Include sample datasets (BTC, ETH, SOL — daily + hourly) and data validation checks

## Implementation Details

### Data Source: Binance Public API

Use Binance's public klines endpoint (no API key required for historical data):

```python
# Base URL for public OHLCV
BASE_URL = "https://api.binance.com/api/v3/klines"

# Parameters:
# symbol: "BTCUSDT", "ETHUSDT", "SOLUSDT"
# interval: "1d" (daily), "1h" (hourly)
# limit: 1000 (max per request)
# startTime/endTime: millisecond timestamps

# Response format: [[open_time, open, high, low, close, volume, close_time, ...], ...]
```

**Rate limits:** 1200 requests/minute (weight-based). Klines is weight=2. Add 100ms sleep between paginated requests to be safe.

**Pagination:** Binance returns max 1000 candles per request. To fetch a full history, paginate using `startTime` set to `last_close_time + 1`.

### DuckDB Schema

```sql
CREATE TABLE IF NOT EXISTS ohlcv (
    symbol      VARCHAR NOT NULL,
    interval    VARCHAR NOT NULL,       -- '1d', '1h'
    timestamp   TIMESTAMP WITH TIME ZONE NOT NULL,
    open        DOUBLE NOT NULL,
    high        DOUBLE NOT NULL,
    low         DOUBLE NOT NULL,
    close       DOUBLE NOT NULL,
    volume      DOUBLE NOT NULL,
    quote_volume DOUBLE,
    num_trades   INTEGER,
    PRIMARY KEY (symbol, interval, timestamp)
);
```

Store the DuckDB file at `machinelearning/snn_applied_finance/backtesting/data/market_data.duckdb`.

### Ingestion Module: `ingest.py`

```python
"""
Key functions:
- fetch_ohlcv(symbol: str, interval: str, start: datetime, end: datetime) -> pl.DataFrame
    Fetches from Binance, returns Polars DataFrame
- store_ohlcv(df: pl.DataFrame, db_path: str) -> int
    Upserts into DuckDB, returns row count
- ingest_symbol(symbol: str, intervals: list[str], start: str, end: str, db_path: str)
    Full pipeline: fetch + validate + store
- ingest_all(symbols: list[str], intervals: list[str], start: str, end: str, db_path: str)
    Batch ingest multiple symbols
"""
```

**Gotchas:**
- Binance returns timestamps in milliseconds — divide by 1000 for Python datetime
- All timestamps must be UTC-aware (`datetime.timezone.utc`)
- Use `httpx` or `requests` with retry logic (3 retries, exponential backoff)
- Polars `pl.from_records()` or `pl.DataFrame()` for ingestion; cast types explicitly

### Loader Module: `loader.py`

```python
"""
Key functions:
- load_ohlcv(symbol: str, interval: str, start: str = None, end: str = None,
             db_path: str = DEFAULT_DB) -> pl.DataFrame
    Load from DuckDB into Polars DataFrame

- to_vectorbt(df: pl.DataFrame) -> vbt_compatible_data
    Convert to VectorBT-compatible format:
    - pandas DataFrame with DatetimeIndex (VectorBT still requires pandas internally)
    - Columns: Open, High, Low, Close, Volume
    - Index: tz-aware DatetimeIndex

- to_nautilus(df: pl.DataFrame, instrument_id: str) -> list[Bar]
    Convert to NautilusTrader Bar objects:
    - from nautilus_trader.model.data import Bar, BarType, BarSpecification
    - from nautilus_trader.model.identifiers import InstrumentId
    - BarSpecification(1, BarAggregation.DAY, PriceType.LAST) for daily
    - Each row becomes a Bar object with proper Nautilus types (Price, Quantity)
"""
```

**Critical NautilusTrader gotcha:** NautilusTrader uses its own `Price` and `Quantity` types, not raw floats. You must construct Bar objects with proper precision:

```python
from nautilus_trader.model.objects import Price, Quantity
from nautilus_trader.model.data import Bar, BarType, BarSpecification
from nautilus_trader.model.enums import BarAggregation, PriceType

# Price precision = 2 for BTCUSDT, 8 for small assets
Price.from_str(str(row["close"]))
Quantity.from_str(str(row["volume"]))
```

### Validation Module: `validation.py`

```python
"""
Key functions:
- validate_no_gaps(df: pl.DataFrame, interval: str) -> ValidationResult
    Check for missing timestamps (allow weekends for crypto? No — crypto trades 24/7)
- validate_timestamps(df: pl.DataFrame) -> ValidationResult
    All timestamps UTC-aware, monotonically increasing, no duplicates
- validate_ohlc_sanity(df: pl.DataFrame) -> ValidationResult
    high >= max(open, close), low <= min(open, close), volume >= 0
- validate_all(df: pl.DataFrame, interval: str) -> list[ValidationResult]
    Run all checks, return summary
"""
```

Test cases:

2. Convert Polars DataFrame → VectorBT format, verify DatetimeIndex + column names
3. Convert Polars DataFrame → NautilusTrader Bars, verify Bar type + count
4. Validation catches: gap detection, bad OHLC (high < low), duplicate timestamps
5. Round-trip: ingest → store → load → validate passes
