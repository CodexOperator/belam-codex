- # Continuous Input Mode Consistently Outperforms Spike Mode *(lessons/continuous-input-beats-spike-mode.md)*
- Across all V2 model variants (SNN and LSTM, all schemes, all sizes), passing raw Gaussian activation values (continuous mode) to the SNN outperforms Bernoulli-sampled spike trains (spike mode). *(lessons/continuous-input-beats-spike-mode.md)*
- **Implication:** For financial SNN architectures, use continuous (graded) input by default. Reserve true spiking input for neuromorphic hardware deployment where it's required. Deterministic evaluation (continuous mode) is essential for reliable metrics — stochastic eval makes all accuracy and Sharpe measurements unreliable. *(lessons/continuous-input-beats-spike-mode.md)*

- Spike-count readout causes dead neurons — always use membrane potential readout snn v4 gradients *(memory/entries/2026-03-17_031637_spike-count-readout-causes-dead-neurons.md)*

- Major Codex Engine V2 coordinate grammar design session with Shael. Finalized dense alphanumeric as the canonical format: e0p3 e1t12 (no separator between namespace+index, spaces only between operations). Ran empirical tokenizer tests: all separators (dot, colon, slash, hyphen) cost same 16 tokens on 4-op chain; dense = 14 tokens; dense numeric = 11 tokens but bad attention signal (model fights arithmetic priors). Alphanumeric wins because token IDs (e=68, p=79, t=83) fire attention patterns associated with symbolic references/identifiers, not arithmetic. Engine mode namespace: e0=orchestrate, e1=edit, e2=create, e3=extend. Flags remain as view modifiers (camera angles) orthogonal to coordinate operations. 'Large alphanumeric model' framing coined by Shael. instance:main codex-engine v2 coordinates grammar tokenizer attention dense-mode alphanumeric *(memory/entries/2026-03-21_065733_major-codex-engine-v2-coordinate-grammar.md)*

Shael framed this as 'a recursive observation of your attention' — architecturally novel and compatible with transformer attention (relational cross-attention between action pattern and target). Key insight: the action IS the map, not documentation about it.
