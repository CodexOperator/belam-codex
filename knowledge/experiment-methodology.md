- # Continuous Input Mode Consistently Outperforms Spike Mode *(lessons/continuous-input-beats-spike-mode.md)*
- Across all V2 model variants (SNN and LSTM, all schemes, all sizes), passing raw Gaussian activation values (continuous mode) to the SNN outperforms Bernoulli-sampled spike trains (spike mode). *(lessons/continuous-input-beats-spike-mode.md)*
- **Implication:** For financial SNN architectures, use continuous (graded) input by default. Reserve true spiking input for neuromorphic hardware deployment where it's required. Deterministic evaluation (continuous mode) is essential for reliable metrics — stochastic eval makes all accuracy and Sharpe measurements unreliable. *(lessons/continuous-input-beats-spike-mode.md)*
- V2 showed population coding (Scheme 0) outperforming delta encoding (Scheme A) by ~2pp accuracy. But population coding uses 35 input neurons (7 features × 5 Gaussian neurons) while delta uses 14 (7 features × 2 channels pos/neg). More inputs → more first-layer capacity → potentially better performance regardless of encoding quality. *(lessons/control-input-dimensionality-in-encoding-comparisons.md)*
- **This is a critical confound.** Until controlled, the statement "population coding outperforms delta encoding" should be qualified as "population coding with higher input dimensionality outperforms delta encoding with lower input dimensionality." *(lessons/control-input-dimensionality-in-encoding-comparisons.md)*
- **General principle:** When comparing encoding schemes, always match input dimensionality or include it as an explicit factor in the experimental design. Otherwise you're comparing encoding quality AND model capacity simultaneously. *(lessons/control-input-dimensionality-in-encoding-comparisons.md)*
